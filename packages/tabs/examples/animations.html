Direction-Aware Slide Animation

Panels slide in from the correct direction based on navigation. Uses --tabs-direction variable (1=forward, -1=backward).

First Panel - Click tabs to the right to see content slide in from the right. Click tabs to the left to see content slide in from the left.
Second Panel - The --tabs-direction variable is set to 1 (forward) or -1 (backward) based on navigation.
Third Panel - This creates a natural carousel-like feel where content flows in the expected direction.
Fourth Panel - The CSS uses calc(var(--tabs-direction) * 30px) to determine slide direction.
Fifth Panel - Try navigating back to the first tab to see the reverse animation.
Direction: -> 0 (initial)
transform: translateX(calc(var(--tabs-direction, 1) * 30px))

Staggered Animation with --tabs-index

Each trigger animates in with a staggered delay based on its --tabs-index. Panels have index-based color accents.

Panel 0 - Each trigger's animation-delay uses calc(var(--tabs-index) * 80ms) for staggered entrance.
Panel 1 - The panel's left border color is calculated from the tab index using HSL.
Panel 2 - Hover over triggers to see index-based background color variations.
Panel 3 - The --tabs-index variable is set on both triggers and panels.
Panel 4 - Refresh the page to see the staggered animation again.
animation-delay: calc(var(--tabs-index, 0) * 80ms) | border-color: hsl(calc(220 + var(--tabs-index) * 20), 70%, 60%)
